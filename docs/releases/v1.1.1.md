## What's Fixed

### Critical: macOS 26 (Tahoe) Crash on Startup

PulseScribe crashed immediately on macOS 26 with:

```
EXC_BREAKPOINT (SIGTRAP)
Must only be used from the main thread
```

**Cause:** macOS 26 introduced stricter thread-safety checks for AppKit. UI updates from background threads (which were tolerated in macOS 15) now cause immediate crashes.

**Fix:** All UI updates are now dispatched to the main thread via `NSOperationQueue.mainQueue()`.

## Changes since v1.1.0

### Bug Fixes

- **Fixed:** Crash on macOS 26 (Tahoe) due to UI updates from background threads
- **Fixed:** Missing loading feedback when model loads on-demand (without preload)
- **Fixed:** Model name not updating after settings change (always showed "Loading turbo...")

### Improvements

- Phase-based loading status ("Loading turbo...", "Warming up...")
- Blue loading animation in overlay (distinct from orange transcribing animation)
- Thread-safe `_update_state()` with automatic main-thread dispatching

## Upgrading from v1.1.0

1. Download the new DMG
2. Replace `/Applications/PulseScribe.app`

Your existing settings in `~/.pulsescribe/.env` are preserved.
